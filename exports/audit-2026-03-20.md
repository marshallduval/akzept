### Format Notes

- **API JSON** is the canonical lossless representation (Zotero Web API v3 internal model)
- **Bibliontology RDF** has the best RDF data model but omits attachments, notes, collections. **Known issue:** As of 2026-03-19, the Bibliontology RDF export fails with HTTP 500 from the Zotero API (see [MODS vs Bibliontology RDF forum](https://forums.zotero.org/discussion/76151/mods-vs-bibliontology-rdf-answered) and [permanent storage export discussion](https://forums.zotero.org/discussion/78254/what-format-to-use-when-exporting-bibliography-for-permanent-storage-in-rep)). This does not affect other export formats or the audit process. **Workaround:** Use CSL JSON, BibTeX, or MODS for archival and interoperability. If Bibliontology RDF is required, consider exporting from Zotero desktop or using Zotero RDF as a fallback.
- **Zotero RDF** includes notes/tags/attachments but uses a poorly-modeled schema (deprecation planned)
- **CSL JSON** is standard for citation interchange but drops non-citation metadata (tags, etc.)
- **BibTeX** has the widest tool compatibility but is lossy and lacks standardization
- **MODS** is a Library of Congress standard; strong for books and library workflows
- **TEI** is the Text Encoding Initiative standard; strong for humanities and text-based scholarship
- **RIS** is a legacy reference manager interchange format; broadly supported

Source: [Zotero Web API v3](https://www.zotero.org/support/dev/web_api/v3/basics), 
[Zotero forum discussion on MODS vs Bibliontology](https://forums.zotero.org/discussion/76151/mods-vs-bibliontology-rdf-answered),
